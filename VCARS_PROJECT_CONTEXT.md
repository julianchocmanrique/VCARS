## 1) What this project is

`VCARS` is a React Native app for workshop intake and service-order flow with role-based UI:

- `administrativo`
- `tecnico`
- `cliente`

The app runs on RN `0.70.6` and currently uses a mixed persistence model:

- Backend API for core vehicle/entry flows.
- `AsyncStorage` still used for session/profile and some local flow state.

## 8) Process state machine (source of truth)

Defined in `vcars/src/lib/vcarsProcess.js` with 6 steps:

1. Recepción (Ingreso)
2. Diagnóstico / Cotización interna
3. Cotización al cliente (Admin)
4. Autorización del cliente
5. Ejecución (Taller)
6. Entrega / Cierre (Admin)

Each step defines:

- `advanceRoles`
- `editableRoles`
- `visibleRoles`

Use helpers from that file instead of hardcoding role/step logic in screens.
